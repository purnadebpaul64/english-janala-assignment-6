Frequently Asked Questions

The difference between var, let, and const

var: var is function-scoped, hoisted with undefined, can be re-declared and updated.


let: let is block-scoped, hoisted but not initialized, can be updated but not re-declared.


const: const is block-scoped, hoisted but not initialized, cannot be updated or re-declared.

The difference between map(), forEach(), and filter()

map() : Returns a new array by applying a function to each element.


forEach(): Iterates over elements but does not return a new array.


filter() : Returns a new array containing only elements that satisfy a given condition.

Explain arrow functions and how they are different from regular functions

Arrow functions offer a shorter syntax and inherit this from the surrounding scope, making them useful in callbacks or event listeners. They don’t have their own this or arguments object, unlike regular functions, which have a dynamic this and access to arguments. Arrow functions also allow implicit returns for single expressions, while regular functions require an explicit return. Arrow functions aren't ideal for object methods or constructors due to the lack of this.

how JavaScript Promises work

JavaScript promise represents an asynchronous operation with three states 1.pending, 2.fulfilled, or 3.rejected. It is created using new Promise(resolve, reject), where resolve() marks success and reject() marks failure.

How closures work in JavaScript

A closure works by allowing an inner function to access variables from its outer function, even after the outer function has finished executing.
